# 语音输入工具(Typeless 风格)设计文档

日期:2026-06-09
状态:已批准设计,待写实现计划

## 目标

一个 Windows 下常驻后台的语音输入工具:按住热键说话,松手后自动完成
语音识别 → LLM 纠错改写 → 把文字注入当前光标处。对标 Typeless,但完全自建、
可控成本与隐私。

核心特性:
- 中英文夹杂语音识别
- 识别后用 LLM 纠错、去除"嗯/啊/那个"等口语填充词
- 跨应用稳定输入

## 已确定的技术选型

| 维度 | 选择 | 理由 |
|------|------|------|
| 运行时 | Rust + Tauri | 体积小、性能好、做成正式产品 |
| ASR | 本地 SenseVoice-Small,经 sherpa-onnx(`sherpa-rs`) | CPU 离线、中英混合开箱即用、纯 Rust 进程 |
| LLM 纠错 | 中转站,OpenAI 兼容 `/v1/chat/completions` | 便宜、可换模型 |
| 计算 | 纯 CPU | 用户机器无独显约束 |
| 热键 | 按住 Win 说话(hold-to-talk) | 用户平时基本不用 Win |
| UI | 托盘图标 + 录音指示浮窗 | 完整体验 |

## 架构

Rust 主进程,内部按职责拆成可独立测试的模块;Tauri 仅承担托盘与指示窗口的"壳"。

```
┌─────────────────────────────────────────────┐
│  Tauri 壳:托盘图标 + 录音指示浮窗 + 设置界面    │
├─────────────────────────────────────────────┤
│  状态机 (Idle→录音→转录→纠错→注入→Idle)        │
├──────────┬──────────┬──────────┬─────────────┤
│ 热键钩子  │ 录音采集  │ ASR引擎   │ LLM纠错      │
│(WH_LL)   │ (cpal)   │(sherpa)  │ (reqwest)    │
├──────────┴──────────┴──────────┴─────────────┤
│  文本注入 (arboard + SendInput Ctrl+V)        │
│  配置 (TOML)                                  │
└─────────────────────────────────────────────┘
```

### 主要依赖(crate)

- `windows`:低级键盘钩子(WH_KEYBOARD_LL)+ SendInput
- `cpal`:麦克风录音
- `sherpa-rs`:SenseVoice ONNX 离线识别
- `reqwest`:HTTP 调用中转站
- `arboard`:剪贴板读写
- `tauri` + 托盘/窗口插件:托盘与浮窗
- `serde` / `toml`:配置

## 模块职责

### 热键钩子 (hotkey)
- 安装 `WH_KEYBOARD_LL` 全局钩子,配独立消息循环线程。
- 行为:
  - **Win 按下**:记录"Win 已按、暂无其他键",keydown 放行,通知开始录音。
  - **按住 Win 期间又按其他键**(Win+E/R/D…):标记组合键 → 放弃本次录音,全部放行,Win 作正常修饰键。
  - **Win 松开**:若全程"单独 Win" → **吞掉 keyup(return 1)** 防止开始菜单弹出,触发转录;若为组合键 → 放行。
  - 按住时长 < ~300ms:仍吞 keyup,但丢弃音频(视为误触)。
- 默认绑左 Win,配置可改右 Win。
- **可测试核心**:把"键盘事件序列 → 动作判定(开始录音/丢弃/触发转录/放行)"抽成纯函数状态机,单测覆盖。

### 录音采集 (audio)
- `cpal` 采集麦克风,16kHz / 单声道 / f32。
- 仅在录音状态缓冲样本;停止时把整段 buffer 交给 ASR。
- 因为是按住说话,不需要 VAD,不需要流式。

### ASR 引擎 (asr)
- `sherpa-rs` 的 OfflineRecognizer 加载 SenseVoice-Small。
- 跑在独立线程(CPU 密集),通过 channel 接收音频、回传文本。
- 模型文件单独存放,配置 `asr.model_dir` 指向模型目录。
- 启动时自检模型存在性。

### LLM 纠错 (corrector)
- POST 到 `{base_url}/chat/completions`,system prompt 负责"去口语填充词 + 纠错 +
  只输出纯文本,保留原意与风格,中英专名保持原样",低 temperature。
- 输出做清理:去掉可能的代码围栏、引号包裹、解释性前后缀。
- **超时(~10s)或失败 → 回退使用 ASR 原文**,保证永不丢输入。
- 可配置:文本短于 N 字时跳过 LLM。
- **可测试核心**:HTTP 用 mock,验证请求构造、输出清理、回退逻辑。

### 文本注入 (inject)
- `paste` 模式(默认):`arboard` 备份当前剪贴板 → 写入文本 → SendInput 发 Ctrl+V →
  延迟 ~80ms → 还原剪贴板。
- `type` 模式(后备):逐字 SendInput 键入,兼容少数拦截粘贴的程序。
- **可测试核心**:注入前的文本清理为纯函数。

### 状态机 (core)
- 状态:`Idle → Recording → Transcribing → Correcting → Injecting → Idle`。
- 驱动托盘图标与指示浮窗的视觉状态。
- 任一环节出错回到 Idle 并反馈。

### Tauri 壳 (ui)
- 托盘图标:idle / 录音中 / 处理中 / 错误 四态。
- 录音指示:小号、置顶、无边框浮窗,显示"🎤 录音中… / ✨ 处理中…"。
- 设置界面:编辑配置(中转站 base_url/key/model、提示词、热键、注入模式等)。

### 配置 (config)
TOML 文件:

```toml
hotkey = "LWin"            # 或 RWin
[asr]
model_dir = "./models/sensevoice"
language  = "auto"         # zh/en/auto
[llm]
base_url = "https://你的中转站/v1"
api_key  = "sk-..."
model    = "gpt-4o-mini"
system_prompt = "..."      # 纠错改写提示词,可改
skip_if_shorter_than = 4   # 短于 N 字跳过 LLM
[inject]
mode = "paste"             # paste | type
```

## 数据流(一次完整输入)

```
按下 Win → 钩子判定"单独 Win" → 开始 cpal 录音(16k/单声道)
  → 说话 …
松开 Win → 钩子吞掉 keyup(防开始菜单) → 停止录音
  → 音频缓冲 → ASR 线程(SenseVoice)→ 原始文本
  → LLM 纠错(去口语 + 纠错)→ 修整文本
  → 备份剪贴板 → 写入文本 → 发 Ctrl+V → 还原剪贴板
  → 回到 Idle
```

## 错误处理

| 情况 | 处理 |
|------|------|
| 无麦克风 / 录音失败 | 托盘错误态 + 系统通知 |
| ASR 模型缺失 | 启动自检,提示配置模型目录 |
| LLM 超时/失败 | 回退使用 ASR 原文,小提示 |
| 钩子安装失败 | 致命,通知并退出 |

## 测试策略

- 单元测试(纯逻辑):热键状态机、配置解析、LLM 纠错(mock HTTP)、注入前文本清理。
- 手动端到端:在记事本、浏览器、聊天软件各验证一遍跨应用注入。
- 音频 / ASR / 钩子本体:隔离纯逻辑后,其余靠手动集成验证。

## 分阶段落地

1. **阶段一(无头核心)**:热键钩子 + cpal 录音 + sherpa ASR + 剪贴板粘贴;不带 LLM、
   不带 UI。打通"按 Win 说话 → 文字出现",这是最大风险的 80%。
2. **阶段二**:加 LLM 纠错 + 配置文件。
3. **阶段三**:Tauri 托盘 + 录音指示浮窗 + 设置界面。

## 非目标(YAGNI)

- 不做流式识别(按住说话不需要)。
- 不做 VAD。
- 不做真正的 TSF 输入法集成(剪贴板粘贴已覆盖绝大多数场景)。
- 不做多语言界面/云同步/账号体系。
