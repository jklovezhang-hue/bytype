# 会议纪要功能 设计文档

**日期**:2026-06-11
**状态**:设计已确认,待写实现计划
**目标**:给 ByType 增加"一键开会录音 → 自动转写 → 区分说话人 → LLM 生成会议纪要"的功能,**完全复用已安装的 SenseVoice 模型**,不引入 Python/PyTorch。

---

## 1. 背景与定位

ByType 是按住热键说话的语音输入工具。会议纪要是"语音输入"的自然延伸(字幕转录则被明确划为视频工具的范畴,不做)。

**核心场景:线上会议为主**(腾讯会议 / Zoom / 飞书)。可采集两个来源:

- **麦克风**(你的声音)
- **系统声音**(对方的声音,WASAPI 环回)

开始会议时**手选录音模式**(麦克风+系统 / 只系统 / 只麦克风,默认麦克风+系统,详见 §3)。两路**分轨**录制带来一个关键优势:麦克风轨天然 = "我",系统轨经说话人分离 = "对方·说话人1/2/3"。

**关键技术结论**(已核实):

- 现用的 `sherpa-rs 0.6` 已自带 `paraformer / silero_vad / diarize / speaker_id / embedding_manager / punctuate` 等模块,会议所需能力齐备,全部走 ONNX Runtime,**无需 Python**。
- SenseVoice **本身不输出时间戳**(只给文本);字幕/会议的时间轴来自 **VAD 分段**(`SpeechSegment { start: i32, samples }`,start 为采样点偏移)。
- **真正的说话人聚类必须"录完整场再处理"**(全程聚类才能让"说话人2"前后一致),因此转写时机锁定为**会后批处理**,不做边录边转。

---

## 2. 范围(YAGNI)

**做**:

- 托盘菜单开始会议 → **弹窗手选录音模式**(麦克风+系统 / 只系统 / 只麦克风);结束会议
- 按模式采集,边录边写盘成 16kHz 单声道 WAV;结束时额外混成 `<base>.wav` 供回放
- **听写共存**:用到麦克风的模式录制期间禁用 ByType 听写,只录系统声音时听写仍可用
- 会后流水线:VAD 分段 → SenseVoice 识别 →(有系统轨则)diarization 分人 → 按时间合并 → 存转写 → LLM 出纪要
- 会议页:录制状态、历史列表、转写+纪要查看、导出/复制/重新生成
- 录音保留策略(默认保留合并录音)+ 磁盘保护 + 崩溃恢复
- 声纹/VAD 模型惰性下载(复用 G5 下载机制)

**不做**(本期):

- 字幕(SRT/VTT)生成 —— 属于视频工具
- 全局快捷键 —— 会议是有意识动作,托盘菜单足够
- 边录边实时转写 —— 与全程 diarization 冲突,且复杂度高
- 线下多人(纯麦克风)优化 —— 架构兼容(降级仅麦克风),但不专门优化
- 给说话人命名/声纹库认人 —— 仅匿名"说话人1/2/3"
- 音频压缩(Opus/FLAC)—— 用 WAV,靠保留策略管磁盘

---

## 3. 架构与模块边界

**核心原则:不动听写热循环。** 会议是一条独立并行路径,与 `engine.rs`(按住说话)解耦,唯一共享 SenseVoice 识别器与 LLM 中转(`corrector`)。

核心 crate 新增 `src/meeting/` 模块组(各司一职、可单测):

| 模块 | 职责 | 依赖 |
|---|---|---|
| `meeting/mod.rs` | `MeetingSession` 编排器:开始/结束、管两路采集、建会议文件夹、跑流水线、状态上报 | 下列全部 |
| `meeting/capture.rs` | 按模式采集(最多两轨):麦克风(cpal,复用现有)→ `mic.wav`;系统环回(`wasapi` crate)→ `system.wav`。就地重采样 16k 单声道,边录边写盘 | cpal + wasapi + 重采样 |
| `meeting/segment.rs` | Silero VAD 把一条轨切成带时间戳(采样点)的语音段 | sherpa-rs silero_vad |
| `meeting/diarize.rs` | 对系统轨聚类出"说话人1/2/3"(惰性加载分段+声纹模型) | sherpa-rs diarize |
| `meeting/transcribe.rs` | 对每段跑 SenseVoice | 复用 `asr::Transcriber` |
| `meeting/minutes.rs` | 合并成"时间+说话人+文字"转写 → 调 LLM 出纪要 | 复用 `corrector` |
| `meeting/store.rs` | 会议文件夹布局、存/读转写与纪要、列历史、保留清理 | — |

**采集方案 = 方案 A**:麦克风沿用现有 cpal(零改动),系统声音用专门的 `wasapi` crate 做 WASAPI loopback。两库各管一轨,**采集阶段分轨不混音**(混音是结束时的后处理步骤,产出 `<base>.wav`);纯 Rust 无 Python。新增依赖:`wasapi`(loopback)、`rubato` 或自写线性重采样(统一到 16k)、`hound` 从 dev 依赖转正(WAV 读写)。

**模型(全部惰性、按需下载,复用 G5 下载机制,存于 `models/` 下)**:

- Silero VAD(~2MB)
- diarization:分段模型(~6MB)+ 声纹模型(几十 MB)

平时纯语音输入完全不加载这些模型;仅在"结束会议 → 处理"阶段加载,处理完释放。

**触发控制 = 仅托盘右键菜单**:空闲时菜单项"开始会议"→ **弹出录音模式选择窗**(见下);确认后开始录制,菜单变"结束会议"(按状态重建托盘菜单)。会议页另放一个停止按钮作顺手后备。无全局快捷键、不碰 LL 钩子与听写状态机。

**Tauri 层**:主窗口新增"会议纪要"导航页(复用主窗口,不新开窗口)。开始会议的模式选择用一个小模态窗(可记住上次选择)。

### 录音模式(开始会议时手动选择)

| 模式 | 录的轨 | 声线处理(diarization) | 录制期间 ByType 听写 |
|---|---|---|---|
| 麦克风 + 系统声音 | `mic` + `system` | ✅ 对 `system` 轨分人 | ❌ 禁用(会议占着麦) |
| 只录系统声音 | `system` | ✅ 对 `system` 轨分人 | ✅ 可用(麦未被占) |
| 只录麦克风 | `mic` | ❌ 不处理(单条 `mic` 轨全标"我") | ❌ 禁用(会议占着麦) |

规则自洽:**声线处理 = 对 `system` 轨做**(有系统声音就处理,没有就不处理);**用到麦克风就禁听写**(避免抢麦),只录系统声音时听写放行(用户还能边开会边用听写记自己的笔记)。

**听写挂起机制**:会议会话与听写引擎同进程,用一个共享 `AtomicBool`(如 `dictation_suspended`)协调——进入用麦的录音模式时置位,引擎循环收到 `StartRecording` 时若已置位则忽略;结束会议复位。不动钩子本身。

## 4. 数据流(端到端)

### ① 开始(托盘"开始会议")

1. **弹窗选录音模式**:麦克风+系统 / 只系统 / 只麦克风(记住上次选择)。
2. 前置检查:磁盘可用空间(< 1GB 警告);**用到麦克风的模式**还需麦克风可用(无则报错拦截)。
3. 建会议文件夹 `<output_dir>/<base>/`(`<base>` = `2026-06-11_143005` 时间戳),记录共同起始时间戳(供混音与时间线对齐)。
4. **按模式启动采集线程**:用麦克风则 cpal → `mic.wav`;录系统则 wasapi 环回 → `system.wav`。各自就地重采样 16k 单声道 16-bit,边录边写盘,内存仅留小缓冲。
5. **用到麦克风的模式 → 置位 `dictation_suspended`**(挂起听写);只录系统则不挂起。
6. 重建托盘菜单为"结束会议";会议页显示"⏺ 录制中 mm:ss"。
7. 若选了录系统但拿不到系统声音(无环回/没在放声):麦克风+系统模式 → 降级为仅麦克风并提示;只录系统模式 → 报错(无可录内容)。

### ② 录制中

持续写盘,不做实时转写。会议页可选显示计时 + 电平条。

### ③ 结束(托盘"结束会议")→ 后台流水线(会议页显示进度,不阻塞 UI)

1. 停采集,定稿 WAV 头;复位 `dictation_suspended`(恢复听写)。
2. **混音**:把本场录到的轨(1 或 2 条)按共同时间线相加 → `<base>.wav`(防削波归一);单轨则直接为该轨内容。
3. **VAD 分段**:Silero 对录到的每条轨切出带时间戳的语音段。
4. **分人**(仅当有 `system` 轨):对 `system` 轨跑 diarization(惰性加载模型)→ 每段标"说话人1/2/3";`mic` 轨全部标"我";只录麦克风时跳过本步,全标"我"。
5. **识别**:SenseVoice 逐段转写(复用 Transcriber)。
6. **合并**:各轨段按起始时间归并成一条时间线 → `[mm:ss] 说话人:文字`。
7. **存转写**:写 `<base>.json`(结构化,供重生成纪要)。
8. **纪要 + 成稿**:带说话人的转写 → LLM(复用 corrector + 会议纪要提示词)→ 写 `<base>.md`(纪要在前 + 转写附录;与 `.wav` 同名)。LLM 失败则 `.md` 仅含转写 + 重试入口。
9. 成功后按 `audio_retention` 决定保留哪些音频(见 §5)。

### ④ 成稿展示

会议页展示转写(可滚动、说话人着色、时间戳)+ 纪要(渲染 Markdown),按钮:复制纪要 / 导出 MD / 重新生成纪要 / 重新处理(留音时)/ 删除会议。

**离线验证**:第 2~7 步是纯批处理,独立成 example `cargo run --example meeting_process -- mic.wav system.wav` → 出 transcript+minutes,不用真采集即可反复验证整条流水线。

## 5. 存储布局与保留策略

**命名约定**:每场会议一个**时间戳命名**的文件夹,夹内所有文件共用同一**时间戳基名** `<YYYY-MM-DD_HHMMSS>`,仅扩展名/次级后缀不同 —— **WAV 与 MD 同名,一眼配对**。

```
<output_dir>/2026-06-11_143005/
├─ 2026-06-11_143005.wav         ← 混音回放(默认保留),16k 单声道
├─ 2026-06-11_143005.md          ← 成稿:纪要 + 转写附录(与 .wav 同名,KB 级)
├─ 2026-06-11_143005.json        ← 结构化转写,供重生成纪要(KB 级)
├─ 2026-06-11_143005.mic.wav     ← 麦克风原始轨(处理用;按档位留删)
└─ 2026-06-11_143005.system.wav  ← 系统声音原始轨(处理用;按档位留删)
```

(只录单一来源时只有对应一条原始轨;`<base>.wav` 即该轨内容。所有录音均为 WAV 格式。)

**整段文件,不切片**(批处理不需要切片;整段最简单、可重处理、崩溃可恢复)。

**大小(关键:采集时即重采样 16k 单声道 16-bit)**:每条 16k 单声道 ~115MB/小时。`<base>.wav` 一条 ~115MB/小时;若同时保留双轨另加 ~230MB/小时。(对比:原始 48k 立体声将达 ~2.8GB/小时,故必须落盘前重采样。)

**保留策略**:转写 + 纪要(KB 级)**永久保留**,从不自动删——这是核心价值。原始/混音音频按 `audio_retention` 三档:

| `audio_retention` | 保留 | 磁盘/小时 | 用途 |
|---|---|---|---|
| `none` | 都删,只剩转写+纪要 | ~0 | 最省 |
| `mixed`(**默认**) | 只留 `<base>.wav` 混音 | ~115MB | 能完整回放整场会 |
| `tracks` | 留 `<base>.wav` + `<base>.mic.wav` + `<base>.system.wav` | ~345MB | 还能重转写/重分人 |

配套:

- 会议页显示"录音占用 X MB" + 一键"清理所有录音"。
- `audio_retention_days`(默认 7):自动删 N 天前的音频(0=永久;transcript/minutes 永不动)。
- **磁盘保护**:开始会议前检查可用空间,< 1GB 警告;录制中写满 → 优雅停止并定稿已录部分。
- **崩溃恢复**:边录边写盘;异常退出后下次启动检测到未完成的会议文件夹 → 提示"从录音恢复处理";WAV 头若未定稿,按文件大小自动修复。

## 6. 配置:新增 `[meeting]` 段

所有字段带 `#[serde(default)]` + `Default`,缺字段不报错(沿用现有 config 风格)。

| 字段 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `output_dir` | String | app 数据区 `meetings/` | 会议根目录,相对路径按现有规则解析 |
| `default_mode` | enum | `mic_system` | 上次/默认录音模式:`mic_system` / `system` / `mic`(开始弹窗预选) |
| `diarization` | bool | `true` | 是否分说话人(有系统轨时生效;关掉则跳过模型下载) |
| `audio_retention` | enum | `mixed` | 音频保留档:`none` / `mixed`(默认,留合并录音) / `tracks`(留双轨) |
| `audio_retention_days` | u32 | `7` | 自动删几天前音频(0=永久;转写/纪要永不删) |
| `minutes_prompt` | String | 空=内置默认 | 会议纪要提示词 |
| `vad_model_url` | String | hf-mirror | Silero VAD 模型下载源 |
| `segmentation_model_url` | String | hf-mirror | diarization 分段模型下载源 |
| `embedding_model_url` | String | hf-mirror | 声纹模型下载源 |

识别语言复用现有 `asr.language`(auto),不新增字段。

**内置会议纪要提示词**(`minutes_prompt` 为空时使用,放 `src/config.rs`):一段中文提示,要求把带说话人和时间的会议转写整理成结构化纪要(议题 / 关键讨论 / 决议 / 待办含责任人时限如有),忠实原意、不编造、只输出纪要正文。

## 7. 错误处理 / 降级(核心目标:至少保住转写)

| 情况 | 处理 |
|---|---|
| 无麦克风(用麦模式) | 开始前明确报错拦截 |
| 拿不到系统声音 | 麦克风+系统模式 → 降级仅麦克风并提示;只录系统模式 → 报错(无可录内容) |
| 磁盘不足 | 开始前警告;录制中写满 → 优雅停止、定稿已录部分 |
| 声纹模型下载/加载失败 | 退化为不分人(系统轨整体标"对方"),仍出转写+纪要,提示 |
| VAD 模型失败 | VAD 为关键件;失败则该步报错并提示重试(不静默降级到固定窗口) |
| 某段识别失败 | 跳过该段、记日志,不拖垮整场 |
| LLM 出纪要失败/超时 | 转写已存盘;页面显示"重试生成纪要" |
| 崩溃/异常退出 | WAV 在盘上;下次启动检测未完成会议 → 提示"从录音恢复处理";WAV 头按文件大小修复 |
| 长会内存 | 全程流式读写盘,绝不整段进内存 |
| 与听写并存 | 由录音模式决定:用麦模式(麦克风+系统 / 只麦克风)录制期间 `dictation_suspended` 置位、听写禁用;只录系统模式听写照常可用。结束会议复位 |
| 只录系统无"我" | 只录系统模式不录麦,转写只含"对方·说话人N";用户如需自己的发言可在该模式下用听写另记 |

## 10. 关键不变量 / 风险

- **WASAPI loopback 是最大未知**:M1 先做半天验证 spike(能从扬声器抓到 PCM)再继续。
- **不动听写热循环与 Win 伪装释放逻辑**:会议路径完全独立。
- **流式落盘,绝不整段进内存**:1 小时双轨 = 230MB,必须边录边写。
